编程结构

编程结构指从程序员的角度来看的处理器结构。主要分为两部分：总线接口部件BIU(Bus Interface Unit)和执行部件EU(Execution Unit).

EU包括ALU、通用寄存器、状态寄存器。执行的是16位的算术与逻辑运算。

通用寄存器即数据寄存器，包括4个16位寄存器AX、BX、CX、DX，其中AX是累加器，每一个均可分为AH和AL两个8位寄存器。

状态寄存器即标志寄存器PSW(Program Status Word)，16位取其中9位作标志位。

8086处理器的状态寄存器通常指的是FLAGS寄存器，它包含了一些标志位，用于存储处理器的状态信息。这些标志位包括：

  1. CF（Carry Flag）：进位标志，若算术运算中最高位发生了进位或借位，置1。

  2. PF（Parity Flag）：奇偶校验标志，若结果的低8位中包含的1的个数是偶数，置1。

  3. AF（Auxiliary Carry Flag）：辅助进位标志，若位3向位4进位或反之，置1，用于辅助BCD运算。

    BCD运算其实就是指对BCD编码后的数进行运算，BCD编码指将十进制每一位数都用四位二进制表示，15的二进制表达是1111，BCD编码是00010101。BCD运算中当每个十进制位上结果大于9，需要减去10，并向高位进1，辅助的标志位就是AF。

  4. ZF（Zero Flag）：零标志，若运算结果各位为零，置1。

  5. SF（Sign Flag）：符号标志，用于指示运算结果的符号，最高位为1表示结果为负数，此时置1；最高位为0表示结果为非负数，此时清0。

  6. TF（Trap Flag）：单步执行标志/陷阱标志位，用于进行调试，当TF为1时，处理器以单步模式执行。(原理是每执行一条指令，进入陷阱中断，执行陷阱中断处理程序)

  7. IF（Interrupt Flag）：中断允许标志，用于控制中断的使能和禁止。置1，处理器可以响应可屏蔽中断。

  8. DF（Direction Flag）：方向标志位，用于控制字符串操作的方向，1表示向低地址方向，即自动减量指令；0表示向高地址方向。

  9. OF（Overflow Flag）：溢出标志，用于指示有无溢出发生。算术运算中带符号的运算超出8位或16位带符号数( 或 )，置1。

其中6、7、8标志位是控制标志，其他的是状态标志.

这些标志位可以被处理器的指令设置、清除或检查，用于判断运算的结果或控制程序的流程。

例：执行5439H+456AH(有符号)的过程中，状态标志结果如下：CF=0; PF=1; AF=1；ZF=0;SF=1; OF=1.

BIU负责存储器与I/O接口之间传送数据，包括段寄存器、指针寄存器、地址加法器和指令队列缓冲器。

段寄存器包括代码段寄存器CS(code segment)、堆栈段寄存器SS(stack segment)、数据段寄存器DS(data segment)、附加段寄存器ES(extra segment).

ES一般和DS功能类似

指针寄存器包括指令指针寄存器IP(instruction pointer)、基地址指针寄存器BP(base pointer)、堆栈指针寄存器SP(stack pointer).

基地址指针寄存器（BP）主要用于指向堆栈帧中的局部变量和函数参数。它通常与堆栈指针寄存器（SP）一起使用，在函数调用和局部变量访问时发挥重要作用。

变址寄存器包括源索引寄存器SI(source index)、目标索引寄存器DI(destination index).大多数时候用于字符串和块传输操作，前者用于指示目标数据(被写入修改或填充)的偏移地址，后者用于指示源数据(被读取复制或提取)的偏移地址。
